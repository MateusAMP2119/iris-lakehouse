# E16.4 Installer ceremony and version gate

Epic: [[Iris Epics]], E16 Install Ceremony and Pipeline Catalog
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

install.sh becomes THE CLI act. TTY branch restaged in the update grammar under a
`── THE CLI ──` chapter mark (bold yellow rule+title, 48-col, after the rainbow
masthead): `→ downloading … / → verifying sha256 … ✓ / → installing <dest> … ✓ /
✓ iris <version> installed`. Non-TTY branch strings stay byte-identical to today.
Transition: `Set up the engine now? (Y/n)` (default yes, /dev/tty, IRIS_FORCE honored)
— the single consent for the binary's acts. Version gate before any offer: probe the
installed binary itself — `"$dest/iris" quickstart --from-installer --json` exit 0 →
exec with the flag; else plain `quickstart --json` exit 0 → exec flagless (E15-era
binary); else no handoff AND no quickstart next-step line (an old binary is never
offered a verb it lacks). Handoff execs the absolute path with `</dev/tty`, temp
cleanup + `trap - EXIT` first; never re-tie stdout. Decline/no-tty/IRIS_FORCE → plain
next-steps (quickstart line gated on the probe; `${dest}/iris` spelled out when dest
not on PATH). README updated (ceremony description, catalog mention). `sh -n` clean;
uninstall.sh untouched.

Spec text (amended Installer handoff Q/A, act ceremony Q/A) landed in E16.1; this task
claims/retargets only.
