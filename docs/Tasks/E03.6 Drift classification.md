# E03.6 Drift classification

Epic: [[Iris Epics]], E03 Declarations, Schemas and Apply
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Implement drift classification as pure diffing logic under the additive-only doctrine: only additive gaps are auto-resolved, removals and non-additive changes are refused outright with no confirmation gate, engine-owned surfaces are excluded from comparison, and cross-mode reads warn without refusing (including in --json output). The table ownership doctrine anchors this cluster as an exempt row.

## Depends on

Epic E01; builds on [[E03.5 Type mapping and DDL]] for parsed table heads and column semantics.

## Contracts (8 testable, 1 exempt)

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S05/drift-additive-only-autofix` | Across schema, ledger, and grant drift, only additive gaps are auto-resolved; every other discrepancy is reported without automatic action. | unit |
| `S05/ledger-drift-removal-refused` | A column removed from table.yaml relative to the migrations ledger is classified non-additive and refused. | unit |
| `S05/schema-drift-excludes-engine-owned` | Capture triggers and the journal are excluded from the schema-drift comparison and are never flagged as drift. | unit |
| `S05/schema-drift-nonadditive-refused` | Schema drift flags an extra, renamed, or retyped live column as non-additive and apply refuses without ever auto-dropping. | unit |
