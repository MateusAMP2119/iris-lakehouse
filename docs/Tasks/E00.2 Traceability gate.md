# E00.2 Traceability gate

Epic: [[Iris Epics]], E00 Conformance Harness and Traceability Gate
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Build the traceability gate itself: recognize contract claims expressed as Go subtest paths or `// spec:` annotations, fail on any non-exempt contract with no claiming test, fail the lint direction on tests claiming no contract, and fail on spec deltas without a test delta. First green build is the gate running over the seeded manifest with every contract listed and every one unclaimed (the full red backlog is the deliverable).

## Depends on

[[E00.1 Manifest seed and doctrine]] (the manifest the gate reads).

## Contracts (4 testable, 0 exempt)

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S16/claims-via-subtest-or-annotation` | The gate recognizes a contract claim expressed either as a Go subtest path or as a // spec: annotation on the test. | unit |
| `S16/gate-fails-unclaimed-contract` | Given a non-exempt contract in the manifest with no test claiming it, the traceability gate fails and reports that contract in the gap list (the TDD backlog). | unit |
| `S16/spec-delta-without-test-fails-gate` | A spec delta (behavioral Q/A added or changed in the doc) without a corresponding test delta fails the traceability gate. | unit |
| `S16/test-without-contract-fails-lint` | A test that claims no contract in the manifest fails the lint direction of the traceability gate (no invented behavior). | unit |

- [ ] Failing tests written at each contract's tier (exempt rows marked exempt in the manifest, no test)
