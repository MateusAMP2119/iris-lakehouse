# E09.5 Route mux and auth

Epic: [[Iris Epics]], E09 Read API, Endpoints and PATs
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Mount the read API on the daemon: the fixed engine-state route roster served as HTTP/1.1 JSON GETs on both the unix socket and the optional TCP listener, the transport auth split (ambient socket vs Bearer token on TCP), per-route scope checks with the 403 split between engine-state and data surfaces, and the exact HTTP status matrix. Also covers role reporting via /healthz and /leader and read service from standby nodes.

## Depends on

Epics E02, E03, E04; builds on [[E09.1 PAT store and scopes]] and [[E09.4 Envelope and serialization]].
