# E07.4 Checkpoint chain and engine key

Epic: [[Iris Epics]], E07 Provenance, Journal Lifecycle and Object Store
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Build the signed checkpoint chain: one journal_checkpoints row per sealed partition, digest hashed over the compacted rows in id order, each digest chained to its parent, signed with the ed25519 engine key minted at install. The digest chain and tamper detection are pure unit logic; journal_checkpoints is insert-only and never pruned so any tampering or loss of sealed history breaks verification visibly.

## Depends on

Epics E05, E06; builds on [[E07.3 Seal and compaction]].
