# E07.5 Object store and export

Epic: [[Iris Epics]], E07 Provenance, Journal Lifecycle and Object Store
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver the content-addressed object store (hash-keyed plain files, immutable, written once) and partition export: a sealed partition exports as one checksummed engine-owned file that round-trips exactly, then the partition is detached, dropped, and its checkpoint flipped to archived. The archive file format round-trips against temp files; meta keeps only hashes and metadata, never payload bytes.

## Depends on

Epics E05, E06; builds on [[E07.4 Checkpoint chain and engine key]].
