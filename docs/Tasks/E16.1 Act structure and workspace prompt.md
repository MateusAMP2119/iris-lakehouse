# E16.1 Act structure and workspace prompt

Epic: [[Iris Epics]], E16 Install Ceremony and Pipeline Catalog
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Restructure the tour into named acts. `tourAct` model (id, title, steps, optional
opener); `quickstartSteps()` → `quickstartActs()` (step ids stable; each step gains an
additive `act` field in the JSON envelope). Chapter marks: `chapterMark` helper — light
rule-and-title (`── THE ENGINE ──…`, 48-col rule, uninstall.sh box family), one palette
color per act (cyan ENGINE, magenta PIPELINE); add the missing `yellow` painter method.
Step progress inside an act uses the `iris update` staged grammar. Consent per act
replaces per-step `Run it? (Y/n)`: one affirmative opens an act, its steps run straight
through; first failing step stops with that command's exit category (existing fault
machinery). ENGINE act opens with `Engine workspace [~/iris]:` — shared-reader
`tourIO.readLine` with visible default, `~` expansion via `os.UserHomeDir`, mkdir -p +
chdir; default is cwd when cwd is already a workspace (`.iris/` or `pipelines/`),
else `~/iris`; `--yes` never prompts and uses cwd (no `./iris-quickstart-demo`; the
concept dies — retire `quickstartDemoDir` and retarget the conformance workspace).
`--from-installer` continuation flag: opens directly on the ENGINE chapter (no welcome,
no act gate — the installer's Y/n was consent); with `--json` stays the inert envelope,
exit 0 (the version-probe guarantee); documented in the handoff Q/A, absent from the
global-flags Q/A. Plain guide restructured under plain-text act headings (golden
re-pinned); PIPELINE act interior keeps the current hardwired hello_iris steps (the
catalog replaces it in E16.2/E16.3).

This task carries the ENTIRE E16 §8 spec delta in one commit set: amended Quickstart
surface Q/A (acts, per-act consent, workspace clause, `--from-installer`), new
Quickstart act ceremony Q/A (artwork contract), amended Installer handoff Q/A (two-act
ceremony, version gate, `Set up the engine now? (Y/n)`), rewritten sample→catalog Q/A
and the interactivity-gating amendment (catalog in the envelope), ALL 10 new E16
contract rows, and the single `spec/inventory.lock` re-record. Later E16 tasks claim
only; they add no spec text.
