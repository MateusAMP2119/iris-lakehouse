# E05.12 Lane runner pass semantics

Epic: [[Iris Epics]], E05 Dispatcher, Lanes and Dead Letters
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Close the epic on the live daemon: perpetual lane runners start eligible pipelines in composer order on every pass, serial within a lane and parallel across lanes, with dispatcher-owned work executing only after a pass completes. Runners read the walk at pass start so graph changes land at pass boundaries, a removed pipeline finishes its current run, each pass starts fresh runs with no auto-retry or backoff, a run failure is never globally fatal, and a hung run holds its lane until the operator cancels it.

## Depends on

Epics E02, E03; builds on [[E05.1 Exec seam]], [[E05.4 Lane model and walk]], and [[E05.5 Gate and consumption]].

| `S01/hung-run-holds-lane` | A hung run holds its lane indefinitely, and the lane resumes only after that run is cancelled by the operator. | integration |
