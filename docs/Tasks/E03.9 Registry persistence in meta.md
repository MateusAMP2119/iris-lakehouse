# E03.9 Registry persistence in meta

Epic: [[Iris Epics]], E03 Declarations, Schemas and Apply
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Persist the declared world into meta against the daemon: the pipelines, dependencies, and lanes table shapes, one atomic dispatcher transaction per apply, composer applies as atomic full-lane rewrites (with single-member lanes producing no row), and the guarantee that resolved env and env_file secret values never land in meta rows. This is the apply-side integration cluster over the E02 daemon.

## Depends on

Epics E01, E02; builds on [[E03.2 Lane composer validation]] and [[E03.4 Dependency graph validation]].
