# E04.4 Connection injection and enforcement

Epic: [[Iris Epics]], E04 Roles, Grants and Credentials
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Close the epic with live enforcement against a running daemon: each pipeline run receives an engine-injected scoped connection for its least-privilege Postgres role at spawn, and database credentials never reach author or consumer hands. Postgres itself is the enforcement point, so an undeclared read fails at the database, and pipeline roles and data PATs are proven unable to reach the meta database.

Epics E02, E03; builds on [[E04.2 Role and credential lifecycle]] and [[E04.3 Grant reconcile and drift]].
