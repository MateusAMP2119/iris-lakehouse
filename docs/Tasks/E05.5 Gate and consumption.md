# E05.5 Gate and consumption

Epic: [[Iris Epics]], E05 Dispatcher, Lanes and Dead Letters
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver the depends_on eligibility gate and run_inputs consumption, pure unit work against the fake process runner. The gate opens on an upstream success not yet consumed, decided by querying run_inputs with no mutable cursor, requires every edge to resolve under fan-in, never imposes execution order or changes walk position, and records the consumed upstream runs write-once at run start. Never-awaited successes are superseded without backlog, and a new edge awaits the upstream's next success rather than consuming history.

## Depends on

Epics E02, E03; builds on [[E05.3 Run records and states]] and [[E05.4 Lane model and walk]].
