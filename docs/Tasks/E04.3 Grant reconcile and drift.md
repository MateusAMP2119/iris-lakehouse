# E04.3 Grant reconcile and drift

Epic: [[Iris Epics]], E04 Roles, Grants and Credentials
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver grant application and reconciliation end to end: apply issues grants on the pipeline's role for exactly the declared tables and fields and records them in meta, the meta access ledger is authoritative with reconciliation emitting grant DDL onto the data database to match it, and grant drift is diffed against the ledger and reconciled. Stray grants exceeding the ledger bounds are classified non-additive, reported, and never silently fixed, and data-PAT grants are recorded per field at mint with no post-mint column creep.

## Depends on

Epics E02, E03; builds on [[E04.1 Access declaration validation]] and [[E04.2 Role and credential lifecycle]].

## Contracts (7 testable, 0 exempt)

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S03/apply-grants-exact-declared` | Apply issues grants on the pipeline's Postgres role for exactly the declared tables and fields, nothing more. | integration |
| `S03/apply-records-access-meta` | Apply records the declared reads/writes entries in meta. | integration |
| `S04/ledger-truth-reconciled` | The access ledger in meta is authoritative and reconciliation emits grant DDL onto the data database to match it. | integration |
| `S05/grant-drift-reconcile` | Grant drift diffs live Postgres grants against the meta access ledger and reconciles so Postgres matches the ledger. | integration |

| `S07/data-pat-no-post-mint-columns` | Grant reconciliation for a data PAT never grants columns added to the table after mint; the diff is computed against the ledger's fixed per-field grant set. | unit |
