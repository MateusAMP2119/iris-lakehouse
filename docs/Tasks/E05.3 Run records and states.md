# E05.3 Run records and states

Epic: [[Iris Epics]], E05 Dispatcher, Lanes and Dead Letters
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver the runs table and the run state machine: a run moves only through queued, running, succeeded, and dead-lettered, and every non-success ending lands in the single dead-lettered terminal state and parks in the dead-letter worklist. Each run row records the declaration checksum, binary hash (artifact_hash null for dev runs), consumed upstream run ids, the data-database LSN pinned at dispatch, and the journal window stamped at dispatch and at the terminal transition.

## Depends on

Epics E02, E03; builds on [[E05.1 Exec seam]].
