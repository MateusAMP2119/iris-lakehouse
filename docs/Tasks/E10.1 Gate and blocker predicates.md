# E10.1 Gate and blocker predicates

Epic: [[Iris Epics]], E10 Destructive Operation Gates
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Implement the pure gate and blocker predicate logic behind the destructive operation gates: the `--yes` versus `--force` semantics (confirmation satisfied versus soft-block override), the soft-block refusals for in-flight runs and un-promoted disposable data, and the destroy blocker rules that scan dependents, run_inputs references, and dead-letter references. All of this is unit-testable decision logic with no daemon required.

## Depends on

Epics E03, E05, E06 (the operations it gates).

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S08/force-overrides-soft-blocks` | On destructive commands `--force` overrides soft-blocks, allowing a soft-blocked operation to proceed. | unit |
| `S08/yes-honors-soft-blocks` | On destructive commands `--yes` satisfies the confirmation prompt but a soft-blocked operation still refuses to proceed. | unit |
