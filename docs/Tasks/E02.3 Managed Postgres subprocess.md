# E02.3 Managed Postgres subprocess

Epic: [[Iris Epics]], E02 Engine Install, Daemon and Leadership
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver managed Postgres as a supervised child subprocess: pinned checksum-verified download into .iris/pg/, subprocess start before any lane dispatch and stop on shutdown, local socket by default, and a fail-fast major-version mismatch guard. External Postgres must drive the exact same admin-DSN code path with no local instance started, keeping managed and external modes on one code path.

## Depends on

Epic E01; builds on [[E02.2 Admin DSN chain]].

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S02/external-pg-identical-path` | With a user-provided pg_dsn the engine starts no local Postgres instance and drives the identical admin-DSN code path as managed mode. | integration |
| `S02/managed-pg-install` | iris engine install downloads the pinned, checksum-verified Postgres into .iris/pg/ with an engine-minted superuser the CLI never sees, using no Docker and no host packages. | conformance |
