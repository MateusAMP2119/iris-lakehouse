# E06.1 Journal DDL and partitioning

Epic: [[Iris Epics]], E06 Write Capture, Wipe and Promotion
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Create the partitioned `public.data_journal` in the data database with the exact declared shape, indexes, and permission posture. This is the storage foundation every other E06 task writes into: id-range partitioning under the configurable `journal_partition_rows` threshold, run windows that never split across partitions, and sealed history that wipe and promote can never touch.

## Depends on

Epics E03, E05.
