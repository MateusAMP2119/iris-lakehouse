# E05.9 Retention and pruning

Epic: [[Iris Epics]], E05 Dispatcher, Lanes and Dead Letters
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver count-based clockless retention with pruning into run_summaries. The pruner keeps the newest retain runs per pipeline (default 1000), writes the archival summary in the same meta transaction that deletes the run, cascades to run_inputs rows and the run's log file, never touches data_journal rows, and spares any dead-lettered run with an outstanding dead_letters entry. run_summaries is insert-only, FK-free, and never pruned.

## Depends on

Epics E02, E03; builds on [[E05.7 Dead letter replay]].
