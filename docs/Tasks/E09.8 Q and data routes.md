# E09.8 Q and data routes

Epic: [[Iris Epics]], E09 Read API, Endpoints and PATs
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Serve the data surface end to end: /q/{endpoint} executing compiled endpoint SQL as the calling PAT's role (endpoints own no roles), /data/{schema}/{table} ad-hoc reads with projection, filters, and keyset paging, and Postgres physically enforcing grant boundaries with 403s that name the endpoint but never the missing fields. Confirms disposable rows come back unfiltered on data-surface reads.

## Depends on

Epics E02, E03, E04; builds on [[E09.6 Endpoint apply lifecycle]] and [[E09.7 Read pool and SQL safety]].

| Contract | Behavior | Tier |
| --- | --- | --- |
| `S04/endpoint-authority-calling-pat` | Endpoints own no roles or credentials; a read request executes under the calling PAT's role. | integration |
| `S07/data-pat-reads-physically-bounded` | Postgres physically bounds a data PAT's reads to its granted fields. | conformance |
| `S07/data-raw-route` | /data/{schema}/{table} serves ad-hoc reads with column projection, eq/range filters, and keyset paging by the table PK. | integration |
