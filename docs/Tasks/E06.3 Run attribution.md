# E06.3 Run attribution

Epic: [[Iris Epics]], E06 Write Capture, Wipe and Promotion
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Attribute every captured write to its run. The run id rides the injected connection as a per-session setting read by the trigger in-transaction, each write produces exactly one journal row committed with the write itself, journal ids per contested row are commit-ordered, and even a dead-lettered run's partial writes stay attributed via its journal_floor/journal_ceiling window.

## Depends on

Epics E03, E05 (run attribution rides the injected connection). Builds on [[E06.2 Capture trigger emission]].
