# E11.2 Standby reads and rejection

Epic: [[Iris Epics]], E11 High Availability and Failover
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Deliver the standby-facing surface: every candidate binds its own listeners and serves reads regardless of role, while mutations against a standby are rejected with exit code 6 and guidance naming the current leader for retargeting. This covers both the listener-level rejection and the shipped-CLI exit code and message shape.

## Depends on

Epics E02, E05; builds on [[E11.1 Leader lock election]] for leadership role state.
